This is the starter site for your product documentation. Replace these pages
with details about your product, your users, and the tasks they need to
complete.

Radiant keeps your docs in this repository, gives your team a visual editor in
the dashboard, and lets you ask the writing agent to help draft or revise pages.
When your changes are ready, publish them to deploy the latest version of your
docs site.

## What to replace first

1. Update the `title` in `docs.json` from `Your Product` to your product name.
2. Rewrite this page so it introduces your product and points readers to the
   most important next step.
3. Replace the example quickstart with the fastest useful workflow for your
   users.
4. Add or remove pages, then update `docs.json` so the navigation matches.

## FAQ

<AccordionGroup>
  <Accordion title="Is this content meant to stay?">
    No. Treat these pages as scaffolding. Keep any structure that helps you,
    then replace the placeholder text with documentation for your own product.
  </Accordion>

  <Accordion title="Do I need to edit files directly?">
    No. You can edit pages, settings, and navigation from the Radiant
    dashboard. If your team prefers source control, you can also edit the MDX
    files and `docs.json` directly.
  </Accordion>

  <Accordion title="What is docs.json?">
    `docs.json` is the configuration file for your docs site. It controls the
    site title, home page, navigation, and other site-level settings.
  </Accordion>

  <Accordion title="Can Radiant help write these docs?">
    Yes. Use the writing agent to draft, revise, reorganize, or clean up docs.
    You can review the changed files before publishing.
  </Accordion>
</AccordionGroup>
