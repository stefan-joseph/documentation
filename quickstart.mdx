Use this page to try the basic Radiant workflow: make a small edit, publish it,
and watch the updated docs deploy.

## Make your first change

### Start small

<Callout type="tip" title="Start with a tiny change">
  Your first edit does not need to be perfect. Change one sentence first so you
  can see how editing, publishing, and deployment work together.
</Callout>

### Follow the publish flow

<Steps>
  <Step title="Edit this page">
    In the Radiant dashboard, open the editor and choose this Quickstart page.
    Change the sentence below to describe your product.

    > Your product helps people accomplish an important task.

    If you prefer source files, edit `quickstart.mdx` directly in this repo.
  </Step>

  <Step title="Notice the file change">
    After you make an edit, Radiant shows a file change indicator in the editor.
    This tells you there are unpublished changes waiting for review.
  </Step>

  <Step title="Publish the change">
    When the draft looks right, click **Publish**. Radiant saves the change to
    the connected docs repo and starts a deployment for the updated site.
  </Step>

  <Step title="Watch the deployment">
    Open **Overview** to see the deployment begin. When the build succeeds, your
    newly published site includes the change you made.
  </Step>
</Steps>

## After the first edit

Once you have seen the workflow, replace the rest of this starter with docs for
your own product.
